什么是工厂

工厂是一个用来创建其它对象的对象。工厂对象通常包含一个或多个方法，用来创建这个工厂所能创建的各种类型的对象。工厂是构造方法的抽象，工厂可以通过各种方式来创建一个对象，并完成复杂的初始化过程或者其它操作。工厂的概念非常广泛：IOC、池化技术甚至单例模式，都有工厂的思想

为什么使用工厂模式

  • 如果创建对象不仅仅只是一个new，而需要一个复杂的过程？比如JDBC的Connection，一系列的准备或者复杂的初始化。如果使用对象就去创建会导致大量重复的代码，我们一定会选择把创建对象的复杂过程封装起来，通过调用来直接获取对象，这个封装，就可以称为工厂，一个为了产生对象的方法。明白了为什么要使用工厂模式，再来理解工厂方法模式就很简单了：工厂方法模式通过定义一个单独的创建对象的方法来解决复杂对象的创建过程。由子类实现这个方法来创建具体类型的对象，把创建对象和对象的类型进行了解耦
  • 工厂方法都有特定的名称，Java中构造方法必须和它所在的类具有相同的名称，如果有多种创建对象的方式就可能导致歧义，我们没有办法通过构造方法名来判断不创建的对象有什么不同。工厂方法可以通过使用具有描述性的名称来使方法的含义非常清晰明了。这时候类的构造方法常常被设置为私有方法，从而强制客户端代码使用工厂方法创建对象
  • 工厂方法可以隐藏创建对象的细节，且不一定每次都会真正创建，完全可以返回缓存的对象
  • 总是引用接口而非实现类，能允许变换子类而不影响调用方，即尽可能面向抽象编程

具体实现

工厂方法

工厂方法，抛开设计模式，它应该是一个方法，具有工厂的作用：产生一个对象

//定义一个对象的抽象
interface Letter {
}
class A implements Letter {
}
class B implements Letter {
}
//定义一个工厂的抽象
interface Factory {
    Letter CreateLetter();
}
//创建对象的具体工厂
class AFactory implements Factory {
    @Override
    public Letter CreateLetter() {
        //大家可以脑补new之前做了茫茫多的工作
        return new A();
    }
}
class BFactory implements Factory {
    @Override
    public Letter CreateLetter() {
        return new B();
    }
}
class Client {
    public static void main(String[] args) {
        BFactory bFactory = new BFactory();
        Letter b = bFactory.CreateLetter();
        
        AFactory aFactory = new AFactory();
        Letter a = aFactory.CreateLetter();
    }
}

普通的工厂方法模式通常伴随着对象的具体类型与工厂具体类型的一一对应，客户端代码根据需要选择合适的具体类型工厂使用。这种选择可能包含复杂的逻辑。这时可以创建一个单一的工厂类，来包含这种选择逻辑，根据参数的不同选择实现不同的具体对象。这个工厂类不需要由每个具体产品实现一个自己的具体的工厂类，所以可以将工厂方法设置为静态方法

简单工厂
interface Letter {
}
class A implements Letter {
}
class B implements Letter {
}
public class FactoryDemo {
    private static Letter getBean(String beanType) {
        Letter result = null;
        if ("A".equals(beanType)) {
            result = new A();
        }
        if ("B".equals(beanType)) {
            result = new B();
        }
        return result;
    }
}
class Client {
    public static void main(String[] args) {
        Letter letter = FactoryDemo.getBean("beanType");
    }
}

简单工厂存在一个问题是，如果传入了一个不存在的beanType，那么可能会返回一个NUll，使用静态工厂方法可以解决这个问题

静态工厂方法
interface Letter {
}
class A implements Letter {
}
class b implements Letter {
}
public class FactoryDemo {
    public static Letter creatA() {
        return new A();
    }
    public static Letter creatB() {
        return new B();
    }
}
class Client {
    public static void main(String[] args) {
        Letter letter = FactoryDemo.creatA();
    }
}

使用范围

  • 创建对象需要大量重复的代码
  • 创建对象需要访问某些信息，而这些信息不应该包含在类中
  • 创建对象的生命周期必须集中管理，以保证在整个程序中具有一致的行为

简单工厂模式并不属于23种设计模式，并且简单工厂模式违反了开闭原则，因如果设计变更，想要让工厂还能够返回一个新的实现类，我们需要添加一段if else这并不友好。抽象工厂可以解决这个问题：工厂需要返回新的实现类的时候，我们不需要修改FactoryDemo的代码
