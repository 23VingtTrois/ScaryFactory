什么是函数式接口？

函数式接口（Functional Interfaces）：如果一个接口只定义了一个抽象方法，那么这个接口就被称为函数式接口。可以包含多个默认方法、类方法，但是只能声明一个抽象方法。同时引入了一个新的注解：@FunctionalInterface。可以把他它放在一个接口前，表示这个接口是一个函数式接口。这个注解的作用和@Override的作用是一样的，这样虚拟机会帮你检查被声明的接口是否符合函数式接口的要求。

下面是一个我们经常用的函数式接口，它只有一个用于比较的方法

@FunctionalInterface
public interface Comparator<T> {
  int compare(T o1, T o2);
}

我们通常会使用匿名内部类的方法来使用这个接口

Collections.sort(personList, new Comparator<PersonBean>() {
             public int compare(PersonBean o1,PersonBean o2) {
                 if(o1.getUserAge().compareTo(o2.getUserAge()) == 0) {
                      return o1.getUesrId().compareTo(o2.getUesrId());
                 }
                 return o1.getUserAge().compareTo(o2.getUserAge());
             }
         }); 

因为Java万物皆对象，所以方法本身不能独立存在，必须要依赖于类。这一点和函数式编程语言不同，所以哪怕我们只是单纯的想要使用某个接口中的一个方法，我们也需要去创建这个接口的实现类，然后去重写这个方法，才可以使用这个方法。虽然我们可以使用匿名内部类来实现接口的方法而不用去特意创建一个接口的实现类，但大佬们还是觉得这样太麻烦了，所以在Java8中Lambda表达式横空出世！

什么是Lambda表达式

Lambda表达式：把函数（就是代码块）作为一个方法的参数（函数作为参数传递进方法中）。通常方法只是接收变量，在方法中使用这个变量，而Lambda表达式是把方法传递进接口中，把传递的方法作为接口的抽象方法的具体实现！Lambda表达式的最终目的是使用更简洁的代码来创建只有一个抽象方法的接口（函数式接口）的实例，来代替匿名内部类的繁琐语法。

下面是compartor用Lambda表达的写法，是不是代码紧凑了很多！

Comparator<PersonBean> soreByAge = (PersonBean x,PersonBean y)-> {
             if(x.getUserAge().compareTo(y.getUserAge()) == 0) {
                 return x.getUesrId().compareTo(y.getUesrId());
             }
             return x.getUserAge().compareTo(y.getUserAge());
         };
Collections.sort(personList, soreByAge);

Lambda表达式的语法

语法组成

  • 形参列表
  • 箭头（->） Java8引入的一个新的操作符，被称为Lamdba操作符，操作符将Lamdba表达式拆分成两部分，左侧是Lamdba的参数列表，右侧是表达式中所需要执行的功能，即Lamdba体
  • Lamdba体

基本语法

  • (parameters) -> expression
  • (parameters) ->{ statements; }

格式一：无参数，无返回值，只有一条语句。
() -> System.out.println("Hello Lambda!");
格式二：有一个参数，且无返回值。若只有一个参数，小括号可以省略不写。
x -> System.out.println(x);
格式三：若 Lambda 体中只有一条语句，return和大括号都可以省略不写
Comparator<Integer> com = (x, y) -> Integer.compare(x, y);
格式四：有两个以上的参数，有返回值，并且 Lambda 体中有多条语句。
Comparator<Integer> com = (x, y) -> {
System.out.println("Hello Lambda!");
return Integer.compare(x, y);
};
格式五：Lambda 表达式的参数列表的数据类型可以省略不写，因为JVM编译器通过上下文推断出数据类型。
(Integer x, Integer y) -> Integer.compare(x, y);
总结

  • 如果方法体里面只有一条语句，那么大括号和return语句都可以不写
  • 参数列表里的参数类型可以不写
  • 如果参数列表只有一个参数，小括号可以不写，但如果没有参数需要加小括号，也就是说，起码的三部分要满足

Lambda的方法引用和构造引用

方法引用(method references)也是在Java8中引入的新特性。若Lambda表达式中的代码块在其他类中已经实现了，我们便可以引用这个类的方法来替代Lamdba。我们可以使用方法引用或构造引用来直接调用传入，可以更进一步的简化代码。

种类语法
静态方法类名::静态方法名
引用某类对象的成员方法类名::成员方法名
引用特定对象的成员方法特定对象::成员方法名
引用构造器类名::new

静态方法和成员方法都可以通过（类名::方法名）来传入

Comparator<PersonBean> soreByAge =PersonBean::compare;

通过（对象::方法名）来传入成员方法，只能传入成员方法

PersonBean person=new PersonBean();
Comparator<PersonBean> soreByAge =person::compare;

通过（类名::new）来传入构造方法

Comparator<PersonBean> soreByAge =PersonBean::new;
