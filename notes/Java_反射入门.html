那么问题来了 Java是怎么把硬盘上的代码加载到内存中呢？这就是Java运行时的第二阶段干的事情，上面流程图的Class类对象阶段。
我们通过使用类加载器（ClassLoader）来把类(class文件)装载进内存，Java中有一个类叫做Class类，这个类用来描述所有的字节码文件。所以在内存中会有一个很多Class类的对象来描述不同的字节码文件，而不管是什么字节码文件，都会由：成员变量（Field）、构造方法（Constructor）、成员方法（Method） 等部分组成，这些会作为Class类的属性。类加载器将Class文件中的成员变量、构造方法、成员方法分别封装为Field、Constructor、Method对象，这就是我们所说的反射机制。

这下面是我自己的理解，对错不负责T^T
我们平时所说的Java万物皆对象，依我个人理解，我们平时所声明的类本质上可以理解Class类对象的实例化。注意是Class类对象的对象。Class类规定了一个类都需要有哪些组成部分，所以我们创建类的时候需要有类名、成员变量、构造方法、成员方法等一系列的组成部分。Class类是所有类的抽象，class关键字声明的类是Class类对象的实例，Class类的实例在内存中描述了一个类是什么样子的，即属性和方法。Java每一个类就对应有一个Class类的对象来描述它。

为什么要用反射

  • 可以在程序的运行过程中操作这些对象。
  • 解耦，提高程序的可扩展性。
  • 极大地提高了代码的灵活性，“动态”的创建对象和调用对象的方法。
    所谓动态，即我们可以通过参数在运行时来选择创建不同的对象或者调用对象的方法，而不是在程序中写死了new 哪一个对象或者是通过对象调用某一个方法。

怎么使用反射

获取Class类对象的实例

首先我们需要获取到某一个类的类对象实例，即Class类的实例，获取一个类对象的实例有三种方法，而这三种方法也分别对应了Java代码在计算机中经历的三个阶段

  1. 源代码阶段
 Class className=Class.forName("包名.类名");
 Class className=Class.forName("Test201908.ReflectObjectPerson");

这个地方一定注意事项全限定名（包名.类名）

  1. Class类对象阶段
 Class className_1= ReflectObjectPerson.class;
  1. Runtime 运行时阶段
ReflectObjectPerson person=new ReflectObjectPerson();
Class className_2= person.getClass();

但是无论我们通过何种方式获取Class类的对象，一个类在一次运行时只会有一个Class类的对象

System.out.println(className.hashCode());
System.out.println(className_1.hashCode());

通过API我们可以看到Class类中的方法大部分都是get方法，我们主要介绍的是关于成员变量、构造方法、和成员方法的获取和使用。

成员变量

我们可以通过成员变量对象来获取、设置成员变量的值

Class className=Class.forName("Test201908.ReflectObjectPerson");     
ReflectObjectPerson reflectObjectPerson=(ReflectObjectPerson) className.newInstance();
//getField返回一个 Field对象，这个对象就是类的一个成员变量，这个成员变量必须是被public修饰的
//如果该类中没有这个字段，会先去找该类所实现的接口中去找，如果接口中没有
//会去该类的父类或者超类中寻找，都没有则抛出NoSuchFieldException
//参数是成员变量的名字，String类型
Field field=className.getField("name");
//set方法的参数是该类的实例和要赋予的值
//第二个参数很好理解，第一个参数后面会详细说明
field.set(reflectObjectPerson,"zhj");

上面这个方法仅适用于获取Public修饰的成员变量，而想要获取public以外的成员变量则需要用下面的方法

//getDeclaredField这个方法和getField的区别是这个方法可以获取到所有的成员变量
 //没有访问权限的限制
 //但只能获取本类中定义的成员变量，不会去寻找接口或父类
Field fieldPrivate =className.getDeclaredField("salary");
//暴力反射，忽略安全访问权限修饰符的安全检查，对Constructor和Method对象的private对象进行操作之前也都需要这一步，否则会报错。
fieldPrivate.setAccessible(true);
fieldPrivate.set(reflectObjectPerson,"200");
构造器方法

成员方法

通过获取成员方法变量来直接运行该方法

Class className=Class.forName("Test201908.ReflectObjectPerson");

//调用成员方法 参数1下面说 参数2是该方法具体参数的值
method.invoke(reflectObjectPerson,"sweets");

以上介绍的方法都是获取了一个成员变量或者一个方法，一个类会有很多成员变量和方法，自然有方法可以获取到所有的，使用方法没有区别，只不过返回值是一个该对象的数组。

Method [ ] methods=className.getMethods();
